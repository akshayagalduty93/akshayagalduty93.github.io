01

Rideshare Driver Income Optimization Analysis

Problem

Rideshare drivers often experience inconsistent income because of downtime, inefficient shift planning, peak-hour dependency, changing demand zones, fuel costs, and limited visibility into earning patterns.

Objective

Identify the key factors affecting driver income and recommend process improvements to increase daily earnings, reduce idle time, and improve utilization.

BA Approach
  • Analysed income patterns across weekdays and weekends.
  • Compared active driving time against waiting or idle time.
  • Identified high-demand time windows and lower-value driving periods.
  • Reviewed operational costs such as fuel, repairs, platform fees, and downtime.
  • Defined KPIs to measure income performance and efficiency.
Key Insights
  • Weekend and peak-hour dependency can create unstable weekly earnings.
  • Idle time directly reduces earnings per hour.
  • Driving in low-demand areas increases waiting time and operational cost.
  • Better shift planning can improve consistency without increasing total work hours.
KPIs Defined
  • Daily Earnings in AUD
  • Earnings per Hour
  • Utilization Rate
  • Idle Time per Shift
  • Fuel Cost as a Percentage of Earnings
Recommendations
  • Plan shifts around high-demand time windows instead of fixed hours.
  • Track earnings per hour to compare profitable and non-profitable shifts.
  • Reduce idle time by choosing stronger demand zones.
  • Use weekly performance tracking to improve driving strategy.
Outcome

Proposed a data-driven shift planning approach that can improve income consistency, reduce wasted time, and support better decision making for rideshare drivers in Melbourne.

02

Login System Requirements & User Flow Analysis

Problem

Users may face login failures, unclear error messages, and poor account recovery options, which can lead to frustration, support requests, and user drop-off.

Objective

Define clear requirements for a simple, secure, and user-friendly login experience.

Functional Requirements
  • User can log in using email and password.
  • User can reset password using the forgot password option.
  • System validates required fields before submission.
  • System displays clear error messages for invalid login attempts.
Non-Functional Requirements
  • Login response time should be under 2 seconds.
  • User credentials should be handled securely.
  • The page should work on desktop and mobile devices.
  • The system should be reliable and available during peak usage.
User Story

As a registered user, I want to log in securely so that I can access my account without issues.

Acceptance Criteria
  • Given valid credentials, when the user clicks login, then the user is redirected to the dashboard.
  • Given invalid credentials, when the user clicks login, then a clear error message is displayed.
  • Given a forgotten password, when the user requests reset, then a reset link is sent to the registered email.
Outcome

Created a clear requirement structure that improves usability, reduces login confusion, and supports better communication between business stakeholders and developers.

03

Customer Support Process Improvement

Problem

Support tickets can reopen after closure when the root cause is not fully resolved, communication is unclear, or closure criteria are not properly defined.

Objective

Improve the support workflow to reduce reopened tickets, improve SLA tracking, and create a clearer resolution process for users and support teams.

BA Approach
  • Reviewed the current support ticket lifecycle from creation to closure.
  • Identified possible reasons for ticket reopening.
  • Defined clearer closure criteria and user confirmation steps.
  • Suggested improvement areas for tracking, communication, and escalation.
Recommendations
  • Add mandatory resolution notes before closing tickets.
  • Introduce user confirmation for selected ticket categories.
  • Track reopened tickets as a separate KPI.
  • Create escalation rules for repeated issues.
Outcome

Proposed a more structured support process to improve service quality, reduce repeated work, and increase customer satisfaction.
